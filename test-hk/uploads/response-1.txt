You don’t win this kind of hackathon by building everything—you win by building the one thing that feels inevitable once people see it.

Right now, most teams will try dashboards + alerts + some AI sprinkled on top. That’s predictable. You want to build something that feels like the future of telco operations.

Let’s sharpen this into a clear, winning direction.

🔥 Core Winning Idea:
“Telco Copilot + Real-Time Network Intelligence Layer”

Think of it as:

“ChatGPT for network engineers + Google Maps for signal quality + Datadog for telcos”

💡 1. Flagship Use Case (Your Differentiator)
🧠 Natural Language Network Intelligence

A network engineer can type:

“Why is Lagos West slow?”
“Show all outages in the last 2 hours affecting 4G users”
“Which tower is causing packet loss in Ikeja?”

And your system:

Interprets intent (via Azure OpenAI + Semantic Kernel)
Queries network logs / metrics
Returns:
Root cause
Affected regions
Suggested fix

👉 This is your wow factor

💡 2. Add a Second Killer Feature (Judges LOVE this)
📍 “Connectivity Intelligence Map” (Public + Internal)

A live map that shows:

Best network zones
Congested areas
Real-time outages
Signal strength heatmap
Two modes:
Engineer mode → Deep diagnostics
Public mode → “Where is MTN fastest right now?”

👉 This ties business value + user impact together

AI summarization
